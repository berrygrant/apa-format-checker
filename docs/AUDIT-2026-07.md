# Audit — July 2026: Latency & Utility

A full audit of the APA 7 thesis checker (server pipeline, rules engine, client, and AWS deployment), followed by fixes for everything listed here. Each finding names the original location; resolutions landed as one commit per phase on this branch.

## Latency findings

| # | Finding | Where it was | Resolution |
|---|---|---|---|
| L1 | Every OpenAI token re-sent the entire accumulated preview over SSE (`{delta, llmPreview}` per token → O(n²) total bytes) | `server/src/lib/jobStore.js` | `llm_delta` now carries only the coalesced `delta` + `previewLength`; 120 ms flush window (`LLM_DELTA_FLUSH_MS`); terminal events flush pending deltas first |
| L2 | LLM prompt duplicated the document ~2× (plain excerpts **and** annotated lines) and was pretty-printed | `server/src/prompts/apaReviewPrompt.js` | Annotated labeled lines only (`"L42: text"` strings), slimmed item issues (cap 80), minified JSON — roughly halves input tokens on the dominant wall-clock stage |
| L3 | All rule sections computed in one synchronous, event-loop-blocking call; the 55%/72% progress stages re-emitted already-computed results in the same tick | `server/src/lib/ruleChecks.js`, `reviewJob.js` | Engine split into exported per-section analyzers; `processReviewJob` interleaves them with `setImmediate` yields so every `section` event streams as it is computed |
| L4 | `writeFileSync`+`renameSync` request-metrics persistence ran in the request path before the SSE stream started | `server/src/lib/requestMetrics.js` | Serialized async persistence queue (`fs/promises`, tmp+rename kept); hot path only enqueues |
| L5 | mammoth/pdf-parse dynamically imported per call — module-load cost landed on the first user review after a cold start | `server/src/lib/docxParser.js` | Cached loaders + `warmParsers()` at server startup; Lambda init absorbs the load; failed warms reset for lazy retry |
| L6 | No OpenAI timeout/retry config (SDK defaults: 10 min, 2 retries) | `server/src/lib/openaiReview.js` | Explicit `OPENAI_TIMEOUT_MS` (240 s) and `OPENAI_MAX_RETRIES` (1); failures still degrade to rule-based-only |
| L7 | CloudFront served the React bundle uncached and uncompressed through Lambda on every page load (single CachingDisabled/`Compress: false` behavior) | `infra/aws/lambda/template.yml` | Dedicated `/assets/*` behavior (CachingOptimized + compression); Express marks hashed assets `immutable, max-age=1y` and `index.html` `no-cache`; SSE default behavior untouched |
| L8 | Client re-rendered the whole tree on every streamed token (zero memoized components; the stream fed a collapsed `<pre>` of raw structured-output JSON) | `client/src/App.jsx`, components | All four child panels memoized with `useCallback`-stabilized handlers; raw-JSON preview replaced by a small "AI review in progress" indicator driven by `previewLength` |

## Utility findings — rule-engine correctness

| # | Finding | Resolution |
|---|---|---|
| U1 | Any parenthetical containing a year counted as a citation — "(the 2008 recession)" became a bogus author-year pair and a false "missing from references" **fail** | Anchored candidate shape (author-like phrase, comma, year, optional locators/year lists/organizational authors); missing-comma citations still count but are flagged |
| U2 | "Smith (2020, p. 5)" was not recognized (regex required `(2020)` exactly) → false "uncited reference" warnings | Narrative pattern accepts locators; sentence subjects like "Table 3 (2020)" are stop-listed |
| U3 | The most common student error "et. al." escaped the malformed-et-al regex | New pattern flags `et. al.`, `et.al`, and bare `et al` while passing correct `et al.` |
| U4 | The references split required a line exactly `references` (first occurrence wins) — "Reference List"/"Bibliography"/"REFERENCES:" dropped the whole list; a TOC entry could hijack the split | Variant labels accepted, last occurrence wins, non-APA labels warn with a rename recommendation |
| U5 | Sentences starting with digits ("5 participants were excluded…") triggered numbered-heading warnings | Candidates must be heading-shaped (≤12 words, title-like start, no sentence punctuation) and ≥2 numbered headings must exist before continuity logic runs |
| U6 | Orphan citations flooded per occurrence (5 repeats = 5 fails + escalation); "2020a" never matched a "2020" reference | Deduplication by source with occurrence counts; escalation uses unique-source counts; year-suffix base-key matching in both directions |
| U7 | Quote-locator warnings fired on scare quotes and quoted titles (any 2 quote characters per paragraph) | Requires a properly paired quotation of ≥8 words |
| U8 | `http://dx.doi.org/…` was flagged as a "bare DOI" | Recognized as URL-formatted; reported as info-level "legacy DOI URL" instead |
| U9 | Title-page check rewarded a "by" line — dissertation-cover style, not APA 7 | Replaced with APA 7 student title-page cues (affiliation, course number, instructor, due date, submission wording); warnings name the missing elements |
| U10 | Heading detection matched only exact names ("Method" but not "Methods"/"Methodology") | Common variants accepted (Methods, Methodology, Materials and Methods, Findings, Background, General Discussion, Conclusions) |
| U11 | Two disagreeing alphabetization implementations; no APA tie-breaks | Single `computeReferenceOrdering` (localeCompare + same-author year tie-break) feeds both the section finding and per-entry issues |
| U12 | "DOCX extraction warning"/"Mammoth" copy shown for PDF uploads | All parser copy is source-aware (DOCX vs PDF) |
| — | Latent bug found while testing: "(e.g., Smith & Lee, 2020)" triggered a false "Non-APA citation format" **fail** (the signal-bare detector matched inside parentheticals) | Signal-bare detection now skips matches inside parenthetical spans |

## Utility findings — reporting & UI

| # | Finding | Resolution |
|---|---|---|
| U13 | No dedup between rule and AI findings; headline counts excluded granular and AI issues entirely; the headline score averaged the rule score with the LLM's self-reported score; three score formulas coexisted | Single severity-weighted formula (`100 − 12·fail − 4·warning − 1·info`) in `scoring.js`; AI duplicates fold into the matching rule item (`alsoFlaggedByLlm`); headline counts derive from the deduplicated inventory; the model's score is reported separately as `summary.aiAssessment`. **Scores shift compared to old reports — intentional.** |
| U14 | Item-level and AI issues were never rendered in the app — only in `.md`/`.docx` exports; students saw ~3–5 fixed findings per section | New "All issues" panel: severity filter chips, section filter, per-issue location labels, recommendations, collapsible excerpts, copy-to-clipboard |
| U15 | The app disclaimed margins/font/spacing/page numbers as unverifiable — but for DOCX those live in the file; raw-text extraction simply discarded them | New layout stage reads `document.xml`/`styles.xml`/headers directly (zero new packages — jszip and @xmldom/xmldom already ship via mammoth): 1-inch margins (metric + binding allowances), APA font list, double spacing, extra space-after, 0.5-inch hanging indents, first-line indents, PAGE fields, Word heading styles, bold/centered title. PDFs keep an honest placeholder |

## Verified non-issues

- Request metrics were already recorded only after upload validation passed.
- SSE heartbeats (15 s) already sit safely inside CloudFront's 60 s origin read timeout.
- The document is parsed exactly once per job.

## Known limitations that remain (by design)

- Layout is measured from **stored** DOCX settings: tab-key indents, manual per-run formatting, and per-page details (widows, exact placement) are invisible.
- PDF uploads get text checks only.
- Reconnecting mid-review restarts the run (no cross-invocation job store).
- Not yet covered: statistics formatting (italic *p*, *M*, *SD*), numbers style, bias-free language, table/figure callouts, seriation — candidates for a future pass now that run-level DOCX data is reachable.
